Do It! Code.org: Create an Animation

 The Challenge

Learn the basics of how to use GameLab and then use your new skills to create a bouncing ball animation.

 Project Steps

  1. Plan your Animation

  2. Create a New Project and Background

  3. Create Your Ball Sprite

  4. Make Your Ball Bounce

Plan your Animation

Before coding your program, it is important that you plan. Use the accordion below to help you make sure you have considered your design before you begin coding. 

Select each item to learn more.

Color of background

What color would you like the background to be? Remember that you can use the W3 Schools Color Webpage to find a color or you can use the lowercase name of a color.

Type of ball to be used

Now think of your ball sprite. Decide what type of ball you would like to use in your animation and whether you would like to draw it yourself or use a pre-drawn one. Do you want it to be a baseball? A fuzzy ball? A soccer ball? You may find it helpful to browse through the animation library for ideas.

Name of sprite variable

Go ahead and decide what you are going to name your sprite variable in your program. A good rule of thumb is to keep it simple with a description such as “bluebaseball” or “basketball” so that you can easily remember it.

Velocity of sprite (x,y)

Finally, decide the velocity of your sprite. Remember that velocity is the speed at which an object travels in a certain direction. Therefore, the larger your numbers are, the farther the sprite will move along the grid in each frame and the faster it will appear to move. If the sprite is too fast it can make the animation disorienting, but if it is too slow it can be boring.

NOTE: The maximum of both the x and y on the grid is 400, so if either number is larger than that you will not be able to see your sprite moving at all.

Create a New Project and Background

Now that you have your animation planned it is time to create it. Make sure that you are using your planning sheet to make things easier. Create a new project in GameLab and rename it. Begin by changing the color of your background. Remember that the function draw { block is an important block for animations, so do not delete it. After you have coded your background to be a different color, run the program to make sure that your code is working so far. If it is not, you need to debug your program to figure out what the problem is.

Create Your Ball Sprite

It is now time to start making your ball. Use your planning sheet to help you. You must begin by finding or creating the animation that you would like to use for your ball. You may want to try searching for “ball” in the animation library.

After you have done that, you can create your ball sprite in your program. Make sure to “run” your program when you are finished. If you have coded your program correctly to make your ball sprite then it should pop up on your grid. If it does not work or does not look the way that you expected, then you may need to debug your program.

NOTE: Recall that any code within the function draw() { block is continuously being redrawn.

Make Your Ball Bounce

Once your program is coded to display your ball sprite correctly you need to make it bounce.  Recall that velocity is the speed of an object in each direction. You need to make sure that your ball has an initial velocity to get it moving. You may find that the velocity you originally planned to use is either too fast or too slow. You can change it as you are coding your program. Once your ball has an initial velocity, it is important that it also has the ability to bounce off the edges of the grid. This ensures that it does not just fly off the grid when it is moving. As you are coding your ball to bounce, run your program to check if it is running correctly. 

HINT: Be aware of which blocks are included in the draw function { block and if they need to be moved out of it.
