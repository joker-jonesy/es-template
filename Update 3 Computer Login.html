Reducing Logins

We heard from many facilitators that having students login to the computer, then login to LearningHub was too time consuming. To reduce the number of logins students have to go through, you can remove the computer login. 

A couple significant things to note regarding this update: 

  1. This change affects how students can save files locally to their computer. Plan to have students save work to a cloud provider such as Google Drive or OneDrive.
  2. This update is only applicable to your SmartLab HQ if you have a SmartLab server and iMacs or Windows machines provided by Creative Learning Systems. This is NOT applicable if your devices are Chromebooks or if learners log into the computers with their district login credentials.

Removing Computer Login 

During your onsite training, you learned how to:

  • Access Active Directory on the server
  • Create a facilitator account in Active Directory
  • Create learner accounts in Active Directory

If you need a refresher on this, please review the Active Directory Refresher below. Otherwise, you can complete the following steps to remove the individual logins from your SmartLab HQ computers.  

  1. Log into it on your Facilitator account on the facilitator computer.

  2. Locate the desktop icon called RDP to CLS-Server. Double-click on the icon and enter the password for einstein (e=mc2). 

  3. Locate the desktop icon called Active Directory. Double-click on the icon.

  4. Create a new learner account by right-clicking on the !Learner Template and selecting Copy.

  5. Create a generic account with username such as Learner and then select Next 

  6. If this asks for a name, using the first name Lifelong and last name Learner will meet this requirement

  7. When setting up no password, make sure you check all options to prevent the password from being changed or expiring.

  8. Review the user account information. If it looks correct, select Finish.

  9. Go onto each SmartLab HQ computer via LanSchool and login as the generic user once.  

  10. Now, when student get to a computer they will type in Learner for the username, and no password will be required.  

Note: you can inactivate the old Active Directories, or leave them as they are, whichever you prefer.

Accessing Active Directory on the Server

For Windows: If you have not yet created a Facilitator account in the Active Directory, you will need to log into einstein on the facilitator computer. If you have a facilitator account set up on Active Directory, log into that account.

  1. To access the server, you need to be logged into einstein on your facilitator computer.
  2. Locate the desktop icon called RDP to CLS-Server. 
  3. Double-click on the icon and enter the password for einstein (e=mc2).
  4. Click OK.
  5. You will only need to access the server to manage Active Directory and Deep Freeze.

For Mac: Log into the Learner account on your facilitator computer.

  1. Locate the desktop icon called Microsoft Remote Desktop.
  2. Double-click on the icon and enter the username and password for einstein.
    1. Username: einstein
    2. Password: e=mc2
  3. Click OK.
  4. You will only need to access the server to manage Active Directory.

Creating Facilitator Accounts

To create facilitator accounts, you will need to use Active Directory on the server.

  1. To access the server, you need to be logged into einstein on your facilitator computer.
  2. Locate the desktop icon called RDP to CLS-Server.
  3. Double-click on the icon and enter the password for einstein: e=mc2.
  4. Locate the desktop icon called Active Directory Users and Computers.
  5. Double-click on the icon.
  6. You will see a list of all current users in the Active Directory. Each facilitator will have his/her own username and password to log into the computer. Active Directory will automatically create a folder in the Facilitator drive that you can use to save, edit, and open files.
  7. To create a new facilitator account, right-click on the !Facilitator Template.
  8. Click on Copy…
  9. Type in your first name and last name.
  10. Type in a “User logon name.” Consider using your school username because it will be easy to remember.
  11. Click “Next.”
  12. Type in a password. Like the username, you could use the same password for your school login account.
  13. Type in the password again to confirm it.
  14. Below the password, there are four account options.
  15. The only box you need to have checked is the one next to “Password never expires.”
  16. Click Next.
  17. Review the user account information. If it looks correct, click Finish.

Creating Learner Accounts

To create learner accounts, you will need to use Active Directory on the server.

  1. Once you have created your own Facilitator account, log into it on your facilitator computer.
  2. Locate the desktop icon called RDP to CLS-Server. 
  3. Double-click on the icon and enter the password for einstein (e=mc2).
  4. Locate the desktop icon called Active Directory.
  5. Double-click on the icon.
  6. You will see a list of all current users in the Active Directory. Each learner will have his/her own username and password to log into the computer. Active Directory will automatically create a folder in the Learners drive that the student can save, edit, and open files.
  7. To create a new learner account, right-click on the !Learner Template.
  8. Click on Copy…
  9. Type in the student’s first name and last name.
  10. Type in the student’s “User logon name.” Consider using the student’s school logon because they already have it memorized.
    1. You can also create a generic account that all students, or all students in a specific grade or class, can use. If doing this, consider using the school’s SmartLab LaunchPad login. This will reduce the number of logins each student has to remember.
  11. Click “Next.”
  12. Type in the student’s password. Like the username, you could assign them the same password for their school login account.
  13. Type in the password again to confirm it.
  14. Below the password, there are four account options.
  15. For middle school and high school learners, you may want to check the box next to “User must change password at next logon.” You can give students a generic password, and then they will be prompted to create their own password when they login.
  16. For the first year, we recommend checking the box next to “User cannot change password.” Sometimes when students have the ability to change their passwords, they end up forgetting what they changed it to and then have to come to the facilitator to get it changed.
  17. For all learners, it is recommended that you keep the box checked next to “Password never expires.”
  18. Uncheck the box next to “Account is disabled.”
    1. Easy way to remember: check the center two items.
  19. Click Next.
  20. Review the user account information. If it looks correct, click Finish.
