Important Vocabulary

  • Energy - the amount of work something could do
  • Algorithm - a specific set of instructions  
  • Numerator - the number above the fraction bar 
  • Denominator - the number below the fraction bar 
  • Unlike Denominators - fractions with denominators that are not the same number
  • Equivalent Fractions - fractions that have different numerators and denominators but represent the same value
  • Least Common Multiple (LCM) - the LCM of two numbers is the smallest number, other than zero, that can be divided by both numbers without leaving a remainder 

What Is a Robot? 

A robot is a type of machine. A machine makes work easier by using a source of energy. A robot is a type of machine, but not all machines are robots. 

Some robots are called electromechanical machines because they use both electronics and mechanics to get work done. Electronics is the science of controlling electrical energy and is needed by ALL robots (and some machines) to do their jobs. Software programs tell robots exactly how to do their jobs and sensors help robots adjust to new situations. Mechanics is the way a robot is built and how it moves. Robots (and all machines) are made up of mechanical parts. Mechanical parts are the physical pieces that make up the machine. Examples include motors, wheels, and gears. 

Sphero Is a Robot

Sphero is a spherical (ball-shaped) robot. Unlike many other robots, Sphero has no outside moving parts. All its electrical and mechanical parts are inside of its hard shell. This allows you to focus on programming.

Coding with Blocks 

One way to program Sphero is with blocks instead of symbols, numbers, or letters. Each block represents a line of code. These blocks of code snap together like a line of puzzle pieces. By connecting blocks of code, you create a computer program. There are different kinds of blocks. Each type comes in a different color. 

To design an LED matrix animation, you will need to create a very simple program using block code.  

Every Sphero block-coding program starts with a black block that says, "on start program." Under the "Lights" tab, you will find a block that says, "matrix animation." When you snap the "matrix animation" block to the "on start program" block, you are telling Sphero to play the matrix animation when the Start button is hit. This is all you need to do if you want to show off your matrix animation. 

If you snap other blocks that follow the matrix animation block, then when the animation is over, Sphero should follow the command on the next block. 

Everyday Algorithms

Algorithms help us to do daily tasks, such as showing how to do the tasks, the order in which to do the tasks, when to start and stop the tasks, and how many times to do the tasks. 

The images below show different examples of how we may follow algorithms in our everyday lives.   

Conditional: IF alarm rings, THEN get up.

Repetition: Brush top teeth 10 times. Brush bottom teeth 10 times. 

Sorting: Is the item paper, plastic, or waste?

Sequencing: Get up FIRST, take a bath  SECOND, and eat THIRD.

Career Connection and Real-World Application

Sphero uses a kind of coding called block programming. Block programming is useful because parts of code are already integrated into each block. Blocks can be rearranged or modified as needed without having to rewrite a lengthy code. Although block programming is often used to teach basic coding to students, computer programmers and software developers may use block programming in their lines of work.

Software developers may use block programming to create new software programs that help users perform certain tasks using different computer devices. 

Computer programmers may use software developed from block programming to solve problems or to improve the functions of a computer.
