What You Should Know - Code Combat: Introduction to JavaScript

Important Vocabulary

  • Inequality - compares two values (examples are the “less than” symbol, <, showing that one value is less than or smaller than the other; and the “greater than” symbol, >, showing that one value is greater or larger than the other)
  • Decimal - a number representing a whole and part of a whole
  • Place Value - the numerical value a digit has based on the position within a number
  • Tenths-Place - the first digit to the right of a decimal
  • Hundredths Place - the second digit to the right of a decimal
  • Thousandths Place - the third digit to the right of a decimal

Writing Code

Like people, computers have their own language they use to communicate. Computers use a written language called source code to work, and humans can tell computers to do things by writing source code. Source code is a language that contains numbers, symbols, and words. In Code Combat, players are able to explore, learn, and practice three different script languages including the popular Python and JavaScript, and an experimental code called CoffeeScript.

Methods

Methods are actions that can be done on objects. In Code Combat, the object is the hero, so methods are actions your hero can do with his gear. Your hero has feet so he can move up, down, right, or left. At the end of each method, there is a parenthesis where an argument can go as shown in the example below. Methods for JavaScript are similar to Python except they have a semicolon at the end of each line, hero.moveLeft(); moves the hero left, hero.moveUp(); moves the hero up, etc.

Arguments and Strings

Arguments in JavaScript language tell the computer more specific details about a method or action. For example, you could tell the computer to move the hero right with hero.moveRight(); or you could use an argument to be more specific about how far you want your hero to move right. You could tell the computer to move farther. If you write hero.moveRight(4);, your hero will move to the right four times.

A string is a piece of text information that is unique to the situation. To tell the computer about a string, surround text with quotation marks.

You can combine arguments and strings by adding quotation marks. For example, you could tell the computer to have the hero attack with hero.attack(enemy) but you must add an argument and string to give your hero specific information about the situation, hero.attack("Treg");.

Variables

Variables are symbols that represent data, but unlike arguments, variables can change. When you are fighting ogres, it's inefficient to type in their names one by one. Imagine if you had to fight a whole army of ogres! If your hero needs to outrun the Sprite and break open a chest, instead of typing their names one by one as strings in an argument, like hero.attack("Treg"), you can use variables to attack the closest ogre and then attack the next closest ogre with findNearestEnemy().

Loops

Sometimes a certain piece of code needs to be repeated over and over. You can use a "while-true" loop and tell the computer to repeat an action however many times you want.

All "while-true" loops in JavaScript must start with a bracket { and end with a bracket }, and each of the steps that you want to be repeated forever while the scenario is true must be indented four spaces under the while (true) words.

Inequality Symbols

Inequalities are symbols used to compare two or more numbers that are not equal. There are two inequality symbols that you will use for this challenge, along with the equals symbol.

Symbol

Description

<

Less than

>

Greater than

=

Equal to

When comparing two numbers you should read from left to right.

For example:
1.6 < 1.9

One and six-tenths is less than one and nine-tenths because the numbers in the ones place are the same but the number in the tenths place are different, and 6 is less than 9.

2.76 > 0.5

Two and seventy-six hundredths is greater than zero and five-tenths because the number in the ones place in two and seventy-six hundredths is higher, or greater.

NOTE: The symbol opens up to whichever value is the largest.

Further details regarding inequalities can be found at Equal, Less, or Greater Than.

Career Connection and Real-World Application

Web Developer

Web developers program websites. They need to know how to code in JavaScript in order to make their websites dynamic; basically, this means that JavaScript is needed for anything interactive like maps, videos, animated graphics, and online shopping.

Many web-based applications also use JavaScript to work, so this would be a good programming language to learn for anyone interested in making apps of all kinds.

Financial Manager

Almost any career that has to make decisions based on analyzing values uses inequalities. Financial managers are responsible for making sure their company meets its financial goals. They generate financial reports that show how money is being earned and spent, and they come up with strategies to better meet their financial goals. To do this well, they need to make sure their company’s earnings are “greater than” their expenses, and they have to regularly calculate how much “greater than” or “less than” to ensure they are always earning enough to make their investors happy.

Water and Sanitation

Even careers you may not think of, like water testers, rely on inequalities to make decisions and recommendations in their profession. Water testers have a very important role, since they test water to make sure that it is safe for people to use, whether that’s for swimming, drinking, watering plants, or other purposes. When water is tested and analyzed, the reports include the test results and the Environmental Protection Agency’s (EPA) safety limits; these limits use inequalities to indicate the safe ranges. For example, for drinking water, the EPA recommends <0.3 parts per million (ppm) of iron. More than 0.3 ppm may give the water a metallic taste and turn the water red. Water testers need to know how to read and interpret these reports.
